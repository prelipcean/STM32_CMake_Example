Clang-Format Configuration Explained

This document explains every setting found in your .clang-format file.

Language

Your Value: Cpp

What it does: Specifies the primary language to be formatted. This tells clang-format to use C++-specific rules.

AccessModifierOffset

Your Value: -2

What it does: The offset (in spaces) for public:, private:, and protected: access modifiers. A negative value "outdents" or "hangs" them, placing them 2 spaces to the left of the current indentation level.

AlignAfterOpenBracket

Your Value: Align

What it does: Controls alignment after an open bracket (, [ or {.

Align: Aligns parameters with the open bracket.

some_function(int a,
              int b);


AlignArrayOfStructures

YourValue: None

What it does: Controls alignment of fields in an array of struct literals. None means no special alignment is performed.

AlignConsecutiveAssignments

Your Value:

Enabled: true

AcrossEmptyLines: true

AcrossComments: true

AlignCompound: false

AlignFunctionDeclarations: false

AlignFunctionPointers: false

PadOperators: true

What it does: This group aligns consecutive assignment operators (=).

Enabled: true: Turns this feature on.

AcrossEmptyLines: true: Keeps alignment even across blank lines.

AcrossComments: true: Keeps alignment even across comments.

AlignCompound: false: Does not align compound assignments (e.g., +=, -=).

AlignFunctionDeclarations: false: Does not align assignments in function declarations.

AlignFunctionPointers: false: Does not align function pointer assignments.

PadOperators: true: Adds spaces to the left of the operator to align, not just the right.

int a   = 1;
int bbb = 2; // 'true' pads before '='


AlignConsecutiveBitFields

Your Value:

Enabled: true

AcrossEmptyLines: true

AcrossComments: true

AlignCompound: false

AlignFunctionDeclarations: false

AlignFunctionPointers: false

PadOperators: false

What it does: This group aligns consecutive bit-field colons (:).

Enabled: true: Turns this feature on.

AcrossEmptyLines: true: Keeps alignment even across blank lines.

AcrossComments: true: Keeps alignment even across comments.

PadOperators: false: Aligns the colons without padding spaces before them.

AlignConsecutiveDeclarations

Your Value:

Enabled: false

AcrossEmptyLines: false

AcrossComments: false

AlignCompound: false

AlignFunctionDeclarations: true

AlignFunctionPointers: false

PadOperators: false

What it does: This group aligns consecutive variable declarations.

Enabled: false: This feature is disabled for regular variables.

AlignFunctionDeclarations: true: This independently enables alignment for consecutive function declarations.

AlignConsecutiveMacros

Your Value:

Enabled: true

AcrossEmptyLines: true

AcrossComments: true

What it does: This group aligns consecutive macro definitions (#define).

Enabled: true: Turns this feature on.

AcrossEmptyLines: true: Keeps alignment even across blank lines.

AcrossComments: true: Keeps alignment even across comments.

#define SHORT_NAME  1
#define LONGER_NAME 2


AlignConsecutiveShortCaseStatements

Your Value: Enabled: false

What it does: When Enabled: true, it would align the statements in consecutive short case blocks. You have this disabled.

(Note: The AlignConsecutiveTableGen... settings are specific to LLVM's TableGen domain-specific language and are all disabled in your file.)

AlignEscapedNewlines

Your Value: Right

What it does: Controls how to align backslashes (\) for escaped newlines (e.g., in macros).

Right: Aligns them to the rightmost column (ColumnLimit).

AlignOperands

Your Value: Align

What it does: Controls alignment of operands in binary expressions.

Align: Aligns operands vertically.

int x = a +
        b +
        c;


AlignTrailingComments

Your Value:

Kind: Always

OverEmptyLines: 0

What it does: Controls the alignment of trailing comments (e.g., // comment).

Kind: Always: Always attempts to align consecutive trailing comments.

OverEmptyLines: 0: Does not align comments that are separated by empty lines.

AllowAllArgumentsOnNextLine

Your Value: true

What it does: If a function call doesn't fit on one line, this allows all arguments to be put on the next line, even if the first one fits.

AllowAllParametersOfDeclarationOnNextLine

Your Value: true

What it does: Similar to the above, but for function declarations.

AllowBreakBeforeNoexceptSpecifier

Your Value: Never

What it does: Never allows a line break between a function's parameter list and its noexcept specifier.

AllowShortBlocksOnASingleLine

Your Value: Never

What it does: Never collapses blocks (e.g., { return 0; }) onto a single line.

AllowShortCaseExpressionOnASingleLine

Your Value: true

What it does: Allows a case statement with a single, short expression to be on one line (e.g., case 1: return;).

AllowShortCaseLabelsOnASingleLine

Your Value: false

What it does: Does not allow case X: labels to be on the same line as the switch statement.

AllowShortCompoundRequirementOnASingleLine

Your Value: true

What it does: In C++ concepts, allows short compound requirements to be on one line (e.g., { x.foo() } -> std::same_as<int>;).

AllowShortEnumsOnASingleLine

Your Value: false

What it does: Does not collapse short enum definitions onto a single line.

AllowShortFunctionsOnASingleLine

Your Value: None

What it does: Never collapses short function definitions onto a single line.

AllowShortIfStatementsOnASingleLine

Your Value: Never

What it does: Never collapses short if statements without braces onto a single line.

AllowShortLambdasOnASingleLine

Your Value: All

What it does: Allows short lambdas of all kinds (with or without braces, empty or not) to be formatted on a single line.

AllowShortLoopsOnASingleLine

YourValue: true

What it does: Allows short for and while loops to be on a single line (e.Example: while (true) continue;).

AllowShortNamespacesOnASingleLine

Your Value: false

What it does: Does not collapse short namespace definitions onto a single line.

AlwaysBreakAfterDefinitionReturnType

Your Value: None

What it does: Never forces a line break after the return type of a function definition.

AlwaysBreakBeforeMultilineStrings

Your Value: false

What it does: Does not force a line break before a multi-line string literal.

AttributeMacros

Your Value: [ __capability ]

What it does: A list of macro names that should be treated like attributes (e.g., [[deprecated]]).

BinPackArguments

Your Value: true

What it does: Tries to pack as many function arguments as possible onto each line (respecting ColumnLimit) instead of putting each on a new line.

BinPackLongBracedList

YourValue: true

What it does: Tries to pack elements in a long braced list (e.g., an array initialization) onto as few lines as possible.

BinPackParameters

Your Value: BinPack

What it does: Tries to pack as many function parameters as possible onto each line in a function declaration.

BitFieldColonSpacing

Your Value: Both

What it does: Adds a space on both sides of the colon in a bit-field declaration (e.g., unsigned a : 1;).

BracedInitializerIndentWidth

Your Value: -1

What it does: A value of -1 means the indent width for braced initializers is not set explicitly and will default to ContinuationIndentWidth (which you have set to 2).

BraceWrapping

Your Value: (Multiple sub-settings)

What it does: This group controls where to put braces ({ and }).

AfterCaseLabel: false: Does not force a brace after a case label.

AfterClass: true: Puts the { for a class definition on a new line (Allman style).

AfterControlStatement: Always: Always puts the { for if, else, for, while on a new line.

AfterEnum: true: Puts the { for an enum on a new line.

AfterExternBlock: true: Puts the { for an extern "C" block on a new line.

AfterFunction: true: Puts the { for a function definition on a new line.

AfterNamespace: true: Puts the { for a namespace on a new line.

AfterObjCDeclaration: true: Puts the { for Objective-C declarations on a new line.

AfterStruct: true: Puts the { for a struct on a new line.

AfterUnion: false: Keeps the { for a union on the same line.

BeforeCatch: true: Puts the { for a catch block on a new line.

BeforeElse: true: Puts the { for an else block on a new line.

BeforeLambdaBody: false: Keeps the { for a lambda body on the same line.

BeforeWhile: false: Keeps the { for a do-while loop's while on the same line.

IndentBraces: false: Does not indent the { and } braces themselves.

SplitEmptyFunction: true: Splits an empty function body ({}) onto two lines.

SplitEmptyRecord: true: Splits an empty class/struct/enum body onto two lines.

SplitEmptyNamespace: true: Splits an empty namespace body onto two lines.

BreakAdjacentStringLiterals

Your Value: true

What it does: Puts adjacent string literals on separate lines.

const char* s = "Hello"
                "World";


BreakAfterAttributes

Your Value: Leave

What it does: Does not change existing line breaks after attributes.

BreakAfterJavaFieldAnnotations

Your Value: false

What it does: (Java-specific) Does not force a break after a field annotation.

BreakAfterReturnType

Your Value: None

What it does: Does not force a line break after a function's return type.

Small-talk:

BreakArrays

Your Value: true

What it does: If an array initializer is too long, this will break it and put each element on a new line if necessary.

BreakBeforeBinaryOperators

Your Value: None

What it does: Breaks lines after binary operators, not before.

int x = a +
        b;


BreakBeforeConceptDeclarations

Your Value: Always

What it does: Always puts a newline before a C++ concept declaration.

BreakBeforeBraces

Your Value: Custom

What it does: Indicates that you are using the detailed BraceWrapping options to control brace placement, rather than a single preset.

BreakBeforeInlineASMColon

YourValue: OnlyMultiline

What it does: In inline assembly, only break before a colon if the statement is multi-line.

BreakBeforeTemplateCloser

Your Value: false

What it does: Does not force a line break before the closing > of a template.

BreakBeforeTernaryOperators

Your Value: true

What it does: Breaks before the ? and : in a ternary expression.

int x = condition
            ? true_value
            : false_value;


BreakBinaryOperations

Your Value: Never

What it does: Does not break binary operations. This is superseded by BreakBeforeBinaryOperators.

BreakConstructorInitializers

Your Value: BeforeColon

What it does: Breaks a constructor initializer list before the colon (:).

BreakFunctionDefinitionParameters

Your Value: false

What it does: Does not force all parameters of a function definition to be on separate lines.

BreakInheritanceList

Your Value: BeforeColon

What it does: In a class definition, breaks the inheritance list before the colon (:).

BreakStringLiterals

Your Value: true

What it does: Allows clang-format to break long string literals.

BreakTemplateDeclarations

Your Value: MultiLine

What it does: Only breaks template declarations onto multiple lines if they are already multi-line.

ColumnLimit

Your Value: 120

What it does: Sets the maximum line length to 120 characters. clang-format will try very hard not to exceed this.

CommentPragmas

What it does: Indents constructor initializer lists by 2 spaces.

ContinuationIndentWidth

What it does: When true, adds spaces inside C++11 braced lists (e.g., MyType{ a, b } instead of MyType{a, b}).

DerivePointerAlignment

Your Value: false

IncludeBlocks

Your Value: Preserve

What it does: This setting is related to include sorting. Preserve means it will keep existing groups of includes separated by empty lines.

IncludeCategories

Your Value: (Multiple categories)

What it 'does': This defines the categories for sorting #include directives.

Priority: 2, Regex: '^"(llvm|llvm-c|clang|clang-c)/': Project headers (LLVM/Clang) are in category 2.

Priority: 3, Regex: '^(<|"(gtest|gmock|isl|json)/)': System headers (<...>) and specific third-party headers (gtest, etc.) are in category 3.

Priority: 1, Regex: '.*': Everything else (which usually means local headers like "my_file.h") is in category 1.

Note: These rules are only applied if SortIncludes: Enabled: is set to true.

IncludeIsMainRegex

IncludeIsMainSourceRegex

Penalty...

What it does: These settings assign "costs" to different formatting choices. clang-format tries to find the lowest-cost formatting.

PenaltyBreakAssignment: 2: Very low cost for breaking after an assignment.

PenaltyBreakBeforeFirstCallParameter: 19: Low cost for breaking before the first parameter in a function call.

PenaltyBreakBeforeMemberAccess: 150: Medium cost for breaking before a . or ->.

PenaltyBreakComment: 300: High cost for breaking lines inside a comment.

PenaltyBreakFirstLessLess: 120: Medium cost for breaking before the first << in a stream insertion.

PenaltyBreakOpenParenthesis: 0: No cost for breaking after an opening parenthesis.

PenaltyBreakScopeResolution: 500: High cost for breaking before a ::.

PenaltyBreakString: 1000: Very high cost for breaking a string literal.

PenaltyExcessCharacter: 1000000: Extremely high cost for any character that goes over ColumnLimit, making it a hard limit.

PenaltyIndentedWhitespace: 0: No cost for using whitespace for indentation.

PenaltyReturnTypeOnItsOwnLine: 1000: Very high cost for putting a function's return type on its own line (discourages it).
